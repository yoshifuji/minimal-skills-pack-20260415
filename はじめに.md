# はじめに - 最小 8 本 Skills 版

この一式は、Web アプリ開発でよく使う機能を **8 本の skill** に絞ってまとめた入口です。

## 最初に把握する 8 本

1. `auth-and-session`
2. `schema-and-migrations`
3. `database-integration`
4. `crud-and-route-handlers`
5. `forms-and-validation`
6. `role-based-access-control`
7. `file-upload-and-storage`
8. `billing-and-payments`

## 読み始める順番

1. `README.md` を読む
2. `docs/機能別Webアプリskills設計書.md` を読む
3. 必要な skill の `SKILL.md` を読む
4. 対応する `references/` を読む

## 迷ったときの選び方

- 会員登録があるなら `auth-and-session`
- DB を使うなら `schema-and-migrations` と `database-integration`
- 一覧、詳細、作成、更新、削除があるなら `crud-and-route-handlers`
- 入力フォームがあるなら `forms-and-validation`
- 管理者や所有者の区別があるなら `role-based-access-control`
- 画像や添付ファイルがあるなら `file-upload-and-storage`
- 支払いがあるなら `billing-and-payments`

## まず作りやすい進め方

1. `auth-and-session`
2. `schema-and-migrations`
3. `database-integration`
4. `crud-and-route-handlers`
5. `forms-and-validation`
6. 必要に応じて 6〜8 本目を追加する

## 迷ったら見る場所

- 認証は `references/認証フロー.md`
- DB 設計は `references/DB変更チェックリスト.md`
- DB 接続は `references/DBアクセスパターン.md`
- 基本操作は `references/基本操作チェックリスト.md`
- フォームは `references/フォームパターン集.md`
- 権限は `references/権限表ひな形.md`
- ファイルは `references/アップロードチェックリスト.md`
- 決済は `references/支払い状態遷移.md`
