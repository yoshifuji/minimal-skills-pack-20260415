# Service Marketplace Skills Pack

Skills プロジェクト説明資料

- 対象: `skills/`, `references/`, `docs/`, `specs/` を使ってアプリを実装する人
- 目的: tutor 固定 pack から、**汎用サービス予約 marketplace pack** へ再構成する
- 想定 stack: Next.js, TypeScript, Supabase, Stripe

---

## このプロジェクトの概要

- この repository は、**provider/customer 型 marketplace を実装するための skills pack** です
- 単なるサンプルコード集ではなく、**設計順序と実装順序** を固定するための pack です
- skills は「何を」「どの順で」「何に注意して」作るかを定義します
- references は、状態表、環境変数、QA、セキュリティ、性能などの補助資料です
- docs は、runbook、setup 手順、運用手順、方針書をまとめる場所です

---

## なぜ再構成したか

- 以前は `tutor-marketplace` 再現用の pack として作られていた
- 汎用 skill と tutor 固有知識が混在していた
- `using-marketplace-agent-skills` が pack 全体の入口になっていた
- `tutor`, `student`, `booking`, `offering` などの語彙が base skill に残っていた
- これを **generic marketplace skill + example overlay** に分離した

---

## repository 構成

- `skills/`
  - 各 skill の本体
  - 1 skill = 1 directory = 1 `SKILL.md`
- `references/`
  - 共通 reference と example reference を置く
- `docs/`
  - 方針書、runbook、手順書、今回の Marp 資料
- `specs/`
  - 実装前に読む仕様入力

## この pack の基本思想

- **spec を先に作る**
- **task を依存順に分解する**
- **schema と auth を payment より先に作る**
- **Checkout より前に reservation を保存する**
- **webhook を source of truth として扱う**
- **refund を UI ではなく money movement として扱う**

## 新しい入口 1: Generic Marketplace

1. `specs/講師マーケットプレイス再現仕様プロンプト.md` を読む
2. `skills/marketplace-foundation-workflow/SKILL.md` を読む
3. `spec-driven-development` で `SPEC.md` を作る
4. `planning-and-task-breakdown` で `tasks/plan.md` と `tasks/todo.md` を作る

注:
- spec ファイル名は互換性のため維持
- 中身は generic なサービス予約 marketplace の example overlay に更新済み

## Codex での skills 登録方法

- Codex 系では、**1 skill = 1 directory + `SKILL.md`** の形で管理する
- 使い方は 2 通り
  - repository 内の `skills/` をそのまま参照させる
  - 個人用 skills として `~/.codex/skills/` 配下へ配置する
- 代表例
  - Windows: `%USERPROFILE%\\.codex\\skills\\<skill-name>\\SKILL.md`
  - macOS / Linux: `~/.codex/skills/<skill-name>/SKILL.md`

---

## Codex での呼び出し方

- prompt で **skill 名を明示**して使う
- 例:
  - `marketplace-foundation-workflow を使って開始してください`
  - `role-based-portals-and-navigation と frontend-ui-engineering を使ってください`
  - `stripe-single-seller-checkout を使って予約と Checkout を実装してください`

実務では、最初に次の 1 行を入れると安定します。

```text
この repository の skills/ と references/ を読み、該当 skill を使って進めてください。
```

## Claude での運用パターン

- 方法 1: repository 参照型
  - Claude に repo を読ませ、`skills/<name>/SKILL.md` を明示的に指定する
- 方法 2: Project Instruction 型
  - 「該当 task に対応する `skills/<name>/SKILL.md` を先に読む」と project 指示に入れる
- 方法 3: prompt template 型
  - skill ごとに定型 prompt を作る

```text
この task では `skills/stripe-single-seller-checkout/SKILL.md` を先に読み、
関連する references も見た上で実装してください。
```
