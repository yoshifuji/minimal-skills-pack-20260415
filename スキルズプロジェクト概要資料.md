# 予約サービス開発用 Skills パック

Skills プロジェクト説明資料

- 対象: `skills/`, `references/`, `docs/`, `specs/` を使ってアプリを作る人
- 目的: 予約・決済を含むアプリ開発を、順序立てて進めやすくする
- 想定する技術構成: Next.js, TypeScript, Supabase, Stripe

---

## この資料でわかること

- この repository で何ができるか
- どの skill をどんな場面で使うか
- 最初に何を読めばよいか
- 実装をどの順で進めると迷いにくいか

---

## この repository の役割

- この repository は、**予約サービス型のアプリ** を作るための skills パックです
- ただコードを並べたものではなく、**進め方そのもの** をまとめています
- skills は「何を」「どの順で」「何に気を付けて」作るかを助けます
- references は、状態表、環境変数、確認項目などの補助資料です
- docs は、手順書や説明資料をまとめる場所です

---

## repository 構成

- `skills/`
  - 各 skill の本体
  - 1 skill = 1 directory = 1 `SKILL.md`
- `references/`
  - 実装時に参照する補助資料
- `docs/`
  - 手順書、説明資料、方針書
- `specs/`
  - 実装前に読む仕様入力

---

## この pack の考え方

- **仕様を先に決める**
- **作業を小さく分ける**
- **DB と認証を先に固める**
- **支払い前に予約データを保存する**
- **Webhook を正しい状態の基準として扱う**
- **返金は見た目ではなく、お金の移動として扱う**

---

## Skills のまとまり 1: 進め方

- `spec-driven-development`
  - 実装前に `SPEC.md` を作る
- `planning-and-task-breakdown`
  - 作業を小さく分ける
- `incremental-implementation`
  - 一度に大きく書かず、小さく積み上げる
- `debugging-and-error-recovery`
  - ビルド失敗や不具合の原因を切り分ける

---

## Skills のまとまり 2: 設計と土台

- `api-and-interface-design`
  - API や型の形を決める
- `nextjs-supabase-ssr-auth`
  - 認証、ログイン状態、役割ごとの遷移を作る
- `postgres-rls-and-public-read-models`
  - DB 定義、RLS、公開用の読み取りモデルを整える
- `security-and-hardening`
  - 入力検証、認可、秘密情報の扱いを固める

---

## Skills のまとまり 3: 予約サービスの実装

- `marketplace-foundation-workflow`
  - どの順で実装するかを整理する
- `role-based-portals-and-navigation`
  - 公開画面、利用者画面、提供者画面を作る
- `stripe-connect-express-onboarding`
  - 提供者の受け取り設定を進める

  - 予約作成から Checkout までをつなぐ
- `payment-webhook-reconciliation`
  - 支払い結果を整合させる
- `refunds-and-cancellation-workflows`
  - キャンセルと返金を扱う

---

## Skills のまとまり 4: 完成前の確認

- `frontend-ui-engineering`

## Codex での使い方

- prompt で skill 名を明示して使う
- 例:
  - `marketplace-foundation-workflow を使って開始してください`
  - `role-based-portals-and-navigation と frontend-ui-engineering を使ってください`
  - `stripe-single-seller-checkout を使って予約と Checkout を実装してください`

実務では、最初に次の 1 行を入れると安定します。

```text
この repository の skills/ と references/ を読み、該当 skill を使って進めてください。
```

## Claude などで使うとき

- repository をそのまま読ませる
- 使いたい skill の `SKILL.md` を先に読むよう指示する
- 必要な reference も一緒に読むようにする

例:

```text
この task では `skills/stripe-single-seller-checkout/SKILL.md` を先に読み、
関連する references も見た上で実装してください。
```
