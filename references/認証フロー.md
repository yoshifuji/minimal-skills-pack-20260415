# 認証フロー

## 基本の流れ

1. 未認証 user が `/signup` または `/login` に入る
2. 認証成功後、session を保存する
3. 認証済み user は保護ページへ進める
4. 未認証 user は保護ページから login へ戻す
5. logout で session を破棄し、未認証状態の画面へ戻す

## 先に決めること

- signup が必要か
- email verify が必要か
- password reset を入れるか
- 認証済み user の初期遷移先
- 未認証 user の戻し先

## 最低限の画面

- `/signup`
- `/login`
- `/forgot-password`
- `/reset-password`

## 最低限の helper

- `getCurrentUser()`
- `requireUser()`
- `redirectIfAuthenticated()`

## 注意

- 認証済み判定を client 側だけに置かない
- secret を使う client は browser に出さない
- session 更新が必要な実装では middleware の責務を先に確認する
