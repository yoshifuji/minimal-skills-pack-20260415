## 基本チェック

### キーボード操作
- [ ] すべての操作要素に Tab キーでフォーカスできる
- [ ] フォーカス順が見た目 / 論理順に従っている
- [ ] フォーカスが見える（フォーカス要素に outline / ring が出る）
- [ ] カスタムウィジェットにキーボード操作がある（Enter で実行、Escape で閉じる）
- [ ] キーボードトラップがない（常に Tab でそのコンポーネントから抜けられる）
- [ ] ページ先頭にスキップリンクがある
- [ ] モーダルは開いている間フォーカスを閉じ込め、閉じたら元の要素に戻す

### スクリーンリーダー
- [ ] すべての画像に `alt` テキストがある（装飾画像は `alt=""`）
- [ ] すべてのフォーム入力に関連するラベルがある（`<label>` または `aria-label`）
- [ ] ボタンとリンクに説明的な文言がある（「ここをクリック」は避ける）
- [ ] アイコンのみのボタンに `aria-label` がある
- [ ] ページに `<h1>` が 1 つあり、見出しレベルを飛ばしていない
- [ ] 動的な内容の変化が通知される（`aria-live` リージョン）
- [ ] テーブルに scope 付きの `<th>` ヘッダーがある

### 視覚面
- [ ] テキストのコントラスト比が 4.5:1 以上（通常テキスト）または 3:1 以上（大きいテキスト、18px 以上）
- [ ] UI コンポーネントのコントラスト比が背景に対して 3:1 以上ある
- [ ] 色だけで情報を伝えていない
- [ ] レイアウトを壊さずにテキストを 200% まで拡大できる
- [ ] 1 秒間に 3 回を超えて点滅するコンテンツがない

### フォーム
- [ ] すべての入力欄に見えるラベルがある
- [ ] 必須項目が示されている（色だけに頼らない）
- [ ] エラーメッセージが具体的で、該当フィールドに関連付いている
- [ ] エラー状態が色以外でも分かる（アイコン、文言、境界線など）
- [ ] フォーム送信エラーが要約され、フォーカス移動できる

### コンテンツ
- [ ] 言語が宣言されている（例: `<html lang="ja">`）
- [ ] ページに説明的な `<title>` がある
- [ ] リンクが周囲のテキストと区別できる（色だけに頼らない）
- [ ] モバイルでタップ領域が 44x44px 以上ある
- [ ] 意味のある空状態がある（真っ白な画面にしない）

## よく使う HTML パターン

### ボタンとリンクの使い分け

```html
<!-- 操作には <button> を使う -->
<button onClick={handleDelete}>タスクを削除</button>

<!-- ナビゲーションには <a> を使う -->
<a href="/tasks/123">タスクを見る</a>

<!-- div/span をボタン代わりにしてはいけない -->
<div onClick={handleDelete}>削除</div>  <!-- 悪い例 -->
```

### フォームラベル

```html
<!-- 明示的な label 関連付け -->
<label htmlFor="email">メールアドレス</label>
<input id="email" type="email" required />

<!-- 暗黙的なラップ -->
<label>
  メールアドレス
  <input type="email" required />
</label>

<!-- 非表示ラベル（見えるラベルを優先） -->
<input type="search" aria-label="タスクを検索" />
```

### ARIA ロール

```html
<!-- ナビゲーション -->
<nav aria-label="メインナビゲーション">...</nav>
<nav aria-label="フッターリンク">...</nav>

<!-- ステータスメッセージ -->
<div role="status" aria-live="polite">タスクを保存しました</div>

<!-- アラートメッセージ -->
<div role="alert">エラー: タイトルは必須です</div>

<!-- モーダルダイアログ -->
<dialog aria-modal="true" aria-labelledby="dialog-title">
  <h2 id="dialog-title">削除の確認</h2>
  ...
</dialog>

<!-- ローディング状態 -->
<div aria-busy="true" aria-label="タスクを読み込み中">
  <Spinner />
</div>
```

### アクセシブルなリスト

```html
<ul role="list" aria-label="タスク">
  <li>
    <input type="checkbox" id="task-1" aria-label="完了: 食材を買う" />
    <label htmlFor="task-1">食材を買う</label>
  </li>
</ul>
```

## クイックリファレンス: ARIA ライブリージョン

| 値 | 挙動 | 用途 |
|-------|----------|---------|
| `aria-live="polite"` | 次の読み上げの切れ目で通知 | ステータス更新、保存完了通知 |
| `aria-live="assertive"` | すぐに通知 | エラー、時間依存の警告 |
| `role="status"` | `polite` と同じ | ステータスメッセージ |
| `role="alert"` | `assertive` と同じ | エラーメッセージ |

## よくあるアンチパターン

| アンチパターン | 問題 | 修正 |
|---|---|---|
| `div` をボタンとして使う | フォーカスできず、キーボード操作もない | `<button>` を使う |
| `alt` テキストがない | 画像がスクリーンリーダーに伝わらない | 説明的な `alt` を付ける |
| 色だけの状態表現 | 色覚多様性のあるユーザーに伝わらない | アイコン、文言、パターンを追加する |
| 自動再生メディア | 混乱を招き、止められない | コントロールを付け、自動再生しない |
| ARIA のないカスタムドロップダウン | キーボードやスクリーンリーダーで使えない | ネイティブ `<select>` か適切な ARIA listbox を使う |
| フォーカスアウトラインの削除 | 今どこにいるか分からなくなる | アウトラインを整えて残す |
| 中身のないリンク / ボタン | 「リンク」とだけ読み上げられ説明がない | 文言または `aria-label` を追加する |
| `tabindex > 0` | 自然なタブ順を壊す | `tabindex="0"` または `-1` だけを使う |
