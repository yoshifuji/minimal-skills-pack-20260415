## コミット前チェック

- [ ] コードに秘密情報がない（`git diff --cached | grep -i "password\|secret\|api_key\|token"`）
- [ ] `.gitignore` が `.env`, `.env.local`, `*.pem`, `*.key` をカバーしている
- [ ] `.env.example` にプレースホルダー値を使っている（実際の秘密情報を入れない）

## 認証

- [ ] パスワードを bcrypt（12 ラウンド以上）、scrypt、または argon2 でハッシュ化している
- [ ] セッションクッキーが `httpOnly`, `secure`, `sameSite: 'lax'` になっている
- [ ] セッションの有効期限が設定されている（妥当な max-age）
- [ ] ログインエンドポイントにレート制限がある（15 分で 10 回以下）
- [ ] パスワード再設定トークンが時間制限付き（1 時間以内）かつ単回使用である
- [ ] 繰り返し失敗した後にアカウントロックする（任意、通知付き）
- [ ] 重要操作で MFA をサポートしている（任意だが推奨）

## 認可

- [ ] すべての保護対象エンドポイントで認証確認をしている
- [ ] すべてのリソースアクセスで所有者 / ロール確認をしている（IDOR 防止）
- [ ] 管理者エンドポイントで admin ロール検証をしている
- [ ] API キーが必要最小限の権限に制限されている
- [ ] JWT トークンを検証している（署名、有効期限、issuer）

## 入力バリデーション

- [ ] すべてのユーザー入力をシステム境界で検証している（API ルート、フォームハンドラ）
- [ ] 検証に allowlist を使っている（denylist ではない）
- [ ] 文字列長を制約している（最小 / 最大）
- [ ] 数値範囲を検証している
- [ ] メール、URL、日付形式を適切なライブラリで検証している
- [ ] ファイルアップロードで型制限、サイズ制限、内容検証をしている
- [ ] SQL クエリがパラメータ化されている（文字列連結しない）
- [ ] HTML 出力をエンコードしている（フレームワークの自動エスケープを使う）
- [ ] リダイレクト前に URL を検証している（open redirect 防止）

## セキュリティヘッダー

```
Content-Security-Policy: default-src 'self'; script-src 'self'
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 0  (無効化し、CSP に依存する)
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

## CORS 設定

```typescript
// 制限的な設定（推奨）
cors({
  origin: ['https://yourdomain.com', 'https://app.yourdomain.com'],
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization'],
})

// 本番で絶対に使わない:
cors({ origin: '*' })  // どのオリジンからでも許可してしまう
```

## データ保護

- [ ] API レスポンスから機密フィールドを除外している（`passwordHash`, `resetToken` など）
- [ ] 機密データをログに出していない（パスワード、トークン、完全なカード番号）
- [ ] 規制で必要なら、PII を保存時暗号化している
- [ ] 外部通信をすべて HTTPS にしている
- [ ] データベースバックアップを暗号化している

## 依存関係のセキュリティ

```bash
# 依存関係を監査
npm audit

# 可能なものは自動修正
npm audit fix

# 重大脆弱性を確認
npm audit --audit-level=critical

# 依存関係を更新し続ける
npx npm-check-updates
```

## エラーハンドリング

```typescript
// 本番: 汎用エラーのみ返し、内部情報は出さない
res.status(500).json({
  error: { code: 'INTERNAL_ERROR', message: '問題が発生しました' }
});

// 本番で絶対にしない:
res.status(500).json({
  error: err.message,
  stack: err.stack,         // 内部情報が漏れる
  query: err.sql,           // データベース詳細が漏れる
});
```

## OWASP Top 10 クイックリファレンス

| # | 脆弱性 | 防止策 |
|---|---|---|
| 1 | Broken Access Control | すべてのエンドポイントで認可確認、所有権検証 |
| 2 | Cryptographic Failures | HTTPS、強いハッシュ、コードに秘密情報を置かない |
| 3 | Injection | パラメータ化クエリ、入力バリデーション |
| 4 | Insecure Design | 脅威モデリング、spec-driven development |
| 5 | Security Misconfiguration | セキュリティヘッダー、最小権限、依存監査 |
| 6 | Vulnerable Components | `npm audit`、依存更新、依存最小化 |
| 7 | Auth Failures | 強いパスワード、レート制限、セッション管理 |
| 8 | Data Integrity Failures | 更新 / 依存を検証し、成果物に署名する |
| 9 | Logging Failures | セキュリティイベントを記録し、秘密情報は記録しない |
| 10 | SSRF | URL を検証 / allowlist し、外向きリクエストを制限する |
