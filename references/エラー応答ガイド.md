# Error Response ガイド

## そろえる項目

- `code`
- `message`
- 必要なら `fieldErrors`

## 例

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容を確認してください",
    "fieldErrors": {
      "email": ["メールアドレスの形式が不正です"]
    }
  }
}
```

## 原則

- user 向け message は平易に書く
- technical detail をそのまま返さない
- endpoint ごとに shape を変えない
