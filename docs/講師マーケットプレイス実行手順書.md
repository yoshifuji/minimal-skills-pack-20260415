# Tutor Marketplace Execution Runbook (Minimal Pack)

この runbook は、generic marketplace skill 群と tutor example overlay を使って `tutor-marketplace` アプリを再現するための実行手順書です。

## 0. 使うファイル

着手時は次の順で読みます。

1. `README.md`
2. `はじめに.md`
3. `skills/example-tutor-marketplace/SKILL.md`
4. `docs/講師マーケットプレイス実行手順書.md`
5. `references/examples/tutor-marketplace/データモデル概要.md`
6. `references/Stripeイベントマトリクス.md`
7. `references/環境変数とシークレット一覧.md`
8. `references/examples/tutor-marketplace/手動QAフロー.md`

## 1. Spec phase

使う skill:
- `spec-driven-development`
- `example-tutor-marketplace`

やること:
- `SPEC.md` を作る
- 画面一覧、API 一覧、DB 一覧、role、booking status、refund status を明記する
- success / cancel / webhook の三系統整合を仕様に書く

完了条件:
- `SPEC.md` があり、auth / RLS / Connect / Checkout / webhook / refund の制約が整理されている

## 2. Plan phase

使う skill:
- `planning-and-task-breakdown`
- `example-tutor-marketplace`

やること:
- `tasks/plan.md` を作る
- `tasks/todo.md` を作る
- 依存順を次に固定する
  1. schema / triggers / RLS
  2. auth / middleware / role redirects
  3. public pages and dashboards
  4. Connect onboarding and sync
  5. booking + Checkout
  6. webhook reconciliation
  7. refunds / cancellation
  8. security hardening
  9. deploy / launch

完了条件:
- タスクが小さく分かれ、各タスクに受け入れ条件がある

## 3. Foundation phase

### 3-1. Database

使う skill:
- `api-and-interface-design`
- `postgres-rls-and-public-read-models`

やること:
- `supabase/schema.sql` を作る
- `profiles`, `tutor_profiles`, `lesson_offerings`, `bookings`, `public_tutor_directory` を定義する
- `updated_at` trigger と `auth.users` trigger を定義する
- index と RLS を入れる

完了条件:
- SQL が一通り書けていて、ownership ルールが明確

### 3-2. Auth / session

使う skill:
- `incremental-implementation`
- `nextjs-supabase-ssr-auth`
- `security-and-hardening`

やること:
- browser / server / admin client を分ける
- `middleware.ts` を置く
- `/signup`, `/login`, logout を作る
- role ごとの redirect を作る
- webhook route を middleware から除外する

完了条件:
- tutor / student 登録とログインが成立する

## 4. Marketplace UI phase

使う skill:
- `frontend-ui-engineering`
- `role-based-portals-and-navigation`

やること:
- `/` で公開講師一覧
- `/tutors/[id]` で講師詳細 + offering 一覧
- `/dashboard/tutor` で onboarding / profile / offering / booking 一覧
- `/dashboard/student` で booking ledger と cancellation 導線
- plain Japanese の status 表現をそろえる

完了条件:
- 匿名、student、tutor で必要画面が出る

## 5. Payments foundation phase

### 5-1. Connect onboarding

使う skill:
- `api-and-interface-design`
- `stripe-connect-express-onboarding`

やること:
- connected account 作成
- onboarding link 発行
- dashboard login link 発行
- status sync route を作る

完了条件:
- tutor dashboard 上で account 作成 → onboarding → sync が通る

### 5-2. Booking + Checkout

使う skill:
- `api-and-interface-design`
- `stripe-single-seller-checkout`

やること:
- student 専用 booking 作成フロー
- pending booking を先に保存
- destination charges で Checkout Session 作成
- success / cancel URL を返す

完了条件:
- student が offering を選び Checkout へ進める

## 6. Reconciliation phase

### 6-1. Webhook

使う skill:
- `payment-webhook-reconciliation`
- `debugging-and-error-recovery`

やること:
- raw body 署名検証
- supported Stripe events を dispatch
- booking / refund / account state を更新
- DB 更新失敗時は 500 を返して Stripe retry を使う

完了条件:
- payment success / failure / expired / account.updated / refund 系が整合する

### 6-2. Refunds

使う skill:
- `refunds-and-cancellation-workflows`
- `security-and-hardening`

やること:
- student 本人のみ future paid booking をキャンセル可能にする
- refund + reverse_transfer + refund_application_fee を実行する
- optimistic patch と webhook 最終整合を両立する

完了条件:
- student dashboard から返金導線が成立する

## 7. Launch phase

使う skill:
- `deployment-for-nextjs-supabase-stripe`
- `shipping-and-launch`

やること:
- `.env.example` を整える
- Supabase URL / redirect URL / Site URL を合わせる
- Stripe webhook endpoint を本番 URL に向ける
- smoke test を実行する

完了条件:
- 本番 URL で signup / onboarding / Checkout / webhook / refund が通る

## 8. 最終チェック

最低限これを確認します。

- `npm run typecheck`
- `npm run build`
- `references/examples/tutor-marketplace/手動QAフロー.md`
- `references/examples/tutor-marketplace/実行成果物チェックリスト.md`

どれかが落ちている場合は、ship 前に戻して修正します。
