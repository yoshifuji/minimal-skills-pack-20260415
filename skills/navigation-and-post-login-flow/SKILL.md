# Navigation And Post Login Flow

## 概要

ログイン後の体験は、単に認証が通るだけでは不十分です。利用者が「次に何をすればよいか」を迷わず理解できる初期遷移、各画面から自然に戻れる導線、role に応じたホームと主要タスクへのショートカットが必要です。

## コアプロセス

1. **role ごとの最初の目的地を決める**
   - `customer` は探す・予約する導線を優先する
   - `provider` はプロフィール整備・プラン管理・予約確認を優先する
   - `admin` は既定 role の近い体験に寄せつつ、将来分岐を残す

2. **ホーム画面の役割を固定する**
   - ダッシュボードは要約と次アクション提示に集中させる
   - 一覧そのものをホームにするか、要約ホームにするかを明示する

3. **成功後リダイレクトを統一する**
   - 作成後は関連一覧か詳細へ戻す
   - 更新後は同じ文脈を保ったまま戻す
   - role と無関係なページへ飛ばさない

4. **戻り先を揃える**
   - 一覧→詳細→編集の往復を自然にする
   - 権限外ページで誤った dashboard に戻さない

5. **主要ナビゲーションを共通化する**
   - role ごとに主要リンクセットを定義する
   - 同じ意味の遷移先が画面ごとにぶれないようにする

6. **パンくずと戻り導線を揃える**
   - 一覧ページにはホーム起点のパンくずを付ける
   - 詳細、作成、設定ページには一覧または親文脈を含むパンくずを付ける
   - 「戻る」はブラウザ履歴ではなく意味的な親画面へ戻す

7. **作成後は次の作業へ着地させる**
   - customer の予約作成後は予約詳細へ遷移する
   - provider のプラン作成後は画像設定や追加設定へ遷移する
   - 完了通知だけで終えず、次アクションへつなげる

## チェックポイント

- [ ] ログイン直後に次の行動がわかる
- [ ] customer と provider で初期導線が明確に異なる
- [ ] 一覧、詳細、作成、編集の戻り先が自然
- [ ] 権限外画面での戻り先が適切
- [ ] 成功後リダイレクトが role と文脈に一致する
- [ ] 一覧、詳細、作成、設定でパンくずが一貫している
- [ ] 共通ヘッダーで主要導線へ常に移動できる
